# 업무 실행기 (Work Executor)
## 완전한 개념 정의서

---

## 🎯 정의

> **업무 실행기란, 어떤 업무 아이디어가 입력되면 그것을 객관화하고, 표준화하고, 결과를 예측하고, 실제로 실행까지 하는 시스템이다.**

---

## 📋 업무 실행기가 하는 7가지

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│                        업무 실행기                               │
│                                                                 │
│   ┌─────────┐                                                   │
│   │  INPUT  │  "휴면고객 재활성화 하고싶어"                        │
│   └────┬────┘                                                   │
│        ↓                                                        │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │  1. 객관화    - 모호한 개념을 명확하게                     │   │
│   │  2. 구체화    - 6W로 실행 가능한 수준까지                  │   │
│   │  3. 표준화    - 글로벌 베스트 프랙티스 적용                 │   │
│   │  4. 예측      - 미래 결과값 계산                          │   │
│   │  5. 실행      - 실제로 작동                               │   │
│   │  6. 측정      - 결과 수집                                 │   │
│   │  7. 개선      - 자동 최적화                               │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│        ↓                                                        │
│   ┌─────────┐                                                   │
│   │ OUTPUT  │  실행 완료 + 결과 리포트 + 개선안                   │
│   └─────────┘                                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 1️⃣ 객관화 (Objectification)

### 하는 일
모호하고 주관적인 아이디어를 **누구나 같은 의미로 이해할 수 있게** 변환

### 예시
```
BEFORE (주관적)          →     AFTER (객관적)
─────────────────────────────────────────────────
"고객 관리 잘 하고 싶어"   →    "30일 미방문 고객에게 재방문 유도 메시지 발송"
"매출 올리고 싶어"        →    "객단가 10% 상승을 위한 번들 상품 추천"
"브랜딩 하고 싶어"        →    "월 4회 인스타그램 콘텐츠 발행"
```

### 방법
- 6W 질문으로 분해
- 측정 가능한 지표로 변환
- 구체적 숫자/기준 부여

---

## 2️⃣ 구체화 (Specification)

### 하는 일
객관화된 개념을 **실제 실행 가능한 수준까지** 상세화

### 6W 프레임워크
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  입력: "휴면고객 재활성화"                                     │
│                                                             │
│  WHO   │ 30일 이상 미방문 고객 (현재 847명)                    │
│  WHAT  │ 10% 할인 쿠폰 + 신규 프로그램 안내                    │
│  WHEN  │ 미방문 30일 도달 시점 자동 발송                       │
│  WHERE │ 카카오톡 알림톡                                      │
│  HOW   │ AI 문구 생성 + 자동 발송                             │
│  WHY   │ 재방문율 15% 목표, 예상 매출 ₩12,700,000             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 출력물
- 실행 가능한 워크플로우
- 필요한 자원 목록
- 일정/타임라인

---

## 3️⃣ 표준화 (Standardization)

### 하는 일
글로벌 최고 수준의 방법론을 **자동으로 적용**

### 적용되는 프레임워크
```
┌─────────────┬──────────────────────────────────────────────┐
│ 단계        │ 적용 프레임워크                               │
├─────────────┼──────────────────────────────────────────────┤
│ 감지        │ Ray Dalio - 약한 신호 포착                    │
│ 분석        │ Elon Musk - 제1원리 사고                      │
│ 전략        │ Peter Thiel - 독점 가능성                     │
│ 설계        │ Jeff Bezos - Working Backwards               │
│ 구축        │ Jeff Bezos - Two-Pizza Team                  │
│ 출시        │ Reid Hoffman - MVP Rule                      │
│ 측정        │ Andy Grove - OKR, Input Metrics              │
│ 학습        │ Ray Dalio - Blameless Post-Mortem            │
│ 확장        │ Jeff Bezos - Flywheel Effect                 │
└─────────────┴──────────────────────────────────────────────┘
```

### 표준화 결과
- 어떤 업무든 9단계로 처리
- 각 단계별 체크리스트 자동 생성
- 글로벌 기준 점수 산출

---

## 4️⃣ 예측 (Prediction)

### 하는 일
실행 **전에** 결과값을 계산해서 알려줌

### 예측 모델
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  📊 미래 결과값 예측                                          │
│                                                             │
│  현재 상태                                                   │
│  ├─ 월 매출: ₩10,000,000                                    │
│  ├─ 성장률: +5%/월                                          │
│  └─ 휴면고객: 847명                                          │
│                                                             │
│  이 업무 실행 시 예측                                         │
│  ├─ 재방문 예상: 127명 (15%)                                 │
│  ├─ 객단가: ₩100,000                                        │
│  ├─ 추가 매출: ₩12,700,000                                  │
│  ├─ 비용: ₩500,000 (메시지 발송)                             │
│  └─ ROI: 2,440%                                             │
│                                                             │
│  외부 요인 반영                                               │
│  ├─ 계절 효과: +2%                                          │
│  ├─ 경쟁 영향: -1%                                          │
│  └─ 최종 예측: ₩12,827,000 (±10%)                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 예측 요소
- 직접 효과 (발송 → 반응 → 전환)
- 간접 효과 (바이럴, 브랜드)
- 외부 요인 (경쟁, 경제, 계절)
- 신뢰 구간 (±범위)

---

## 5️⃣ 실행 (Execution)

### 하는 일
계획을 **실제로 작동**시킴

### 실행 유형
```
┌─────────────┬──────────────────────────────────────────────┐
│ 유형        │ 설명                                         │
├─────────────┼──────────────────────────────────────────────┤
│ 자동 실행   │ 승인 없이 바로 실행 (트리거 기반)              │
│ 1클릭 실행  │ 확인 버튼 하나로 실행                         │
│ 검토 후 실행│ 담당자 검토 → 승인 → 실행                     │
└─────────────┴──────────────────────────────────────────────┘
```

### 실행 범위
- 메시지 발송 (카카오, SMS, 이메일, 푸시)
- 콘텐츠 생성 (텍스트, 이미지, 영상)
- 데이터 처리 (세그먼트, 필터, 추출)
- 외부 연동 (CRM, 결제, 예약)

---

## 6️⃣ 측정 (Measurement)

### 하는 일
실행 결과를 **자동으로 수집하고 분석**

### 측정 지표
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  📈 실행 결과 리포트                                          │
│                                                             │
│  실행 지표                                                   │
│  ├─ 발송: 847건                                             │
│  ├─ 도달: 823건 (97.2%)                                     │
│  ├─ 오픈: 412건 (50.1%)                                     │
│  └─ 클릭: 156건 (37.9%)                                     │
│                                                             │
│  결과 지표                                                   │
│  ├─ 재방문: 89명 (10.5%)                                    │
│  ├─ 구매: 67명 (7.9%)                                       │
│  └─ 매출: ₩8,900,000                                        │
│                                                             │
│  예측 vs 실제                                                │
│  ├─ 예측: ₩12,700,000                                       │
│  ├─ 실제: ₩8,900,000                                        │
│  └─ 달성률: 70.1%                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 측정 원칙
- 예측값과 실제값 비교
- Input 지표 (통제 가능) 우선
- 실시간 대시보드

---

## 7️⃣ 개선 (Improvement)

### 하는 일
측정 결과를 바탕으로 **다음 실행을 자동 최적화**

### 개선 루프
```
     측정 결과
         │
         ↓
    ┌─────────┐
    │ 분석    │ 왜 예측과 달랐나?
    └────┬────┘
         │
         ↓
    ┌─────────┐
    │ 학습    │ 어떤 요소가 영향을 미쳤나?
    └────┬────┘
         │
         ↓
    ┌─────────┐
    │ 적용    │ 다음 실행에 반영
    └────┬────┘
         │
         ↓
    다음 실행 (개선된 버전)
```

### 개선 항목
- 발송 시간 최적화
- 메시지 문구 A/B 테스트
- 타겟 세그먼트 조정
- 채널 믹스 변경
- 예측 모델 보정

## ✅ 업무 실행기의 가치

| 없을 때 | 있을 때 |
|--------|--------|
| "뭘 해야 할지 모르겠어" | 명확한 다음 액션 |
| "이게 맞는 방법인지..." | 글로벌 표준 적용됨 |
| "효과가 있을까?" | 예측 결과값 제시 |
| "내가 직접 해야 해" | 자동 실행 |
| "결과를 모르겠어" | 실시간 측정 |
| "다음엔 어떻게?" | 자동 개선안 제시 |
