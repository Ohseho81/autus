## 🎯 핵심 개념 정리

### 1. 워크플로우 구조: 선형 + 부분 순환

```
[이벤트 입력] → [이벤트 정의] → [6W 명확화] → [9단계 실행] → [결과]
                                                    ↓
                                              [반복 이벤트만]
                                                    ↓
                                              [자동 재실행]
```

- **전체 순환 X** → 명확한 시작점 (이벤트 입력)
- **부분 순환 O** → 반복 이벤트만 측정→감지로 피드백

## 📝 PHASE 0: 이벤트 입력 & 정의

### 0-1. 이벤트 워딩 입력
사용자가 자유롭게 이벤트/아이디어를 텍스트로 입력

```
예시:
- "3개월 훈련 하이라이트 영상 제공"
- "휴면 고객 재활성 캠페인"
- "신규 회원 첫 구매 유도"
```

### 0-2. 이벤트 유형 정의

| 구분 | 단발성 (One-time) | 반복성 (Recurring) |
|-----|------------------|-------------------|
| 정의 | 1회 실행 후 종료 | 트리거 조건 충족 시 자동 반복 |
| 예시 | 연말 프로모션, 오픈 기념 | 생일 축하, 휴면 고객, 재구매 유도 |
| 워크플로우 | 선형 (시작→끝) | 순환 (측정→감지→재실행) |

### 0-3. 6하원칙으로 워딩 구체화

**입력된 워딩을 기반으로 객관식 질문 생성**

```
입력: "3개월 훈련 하이라이트 영상 제공"

→ WHO (누구에게):
  □ 전체 회원  □ 3개월 이상 회원  □ 프리미엄 회원  □ 직접 입력

→ WHAT (무엇을):
  □ 개인 맞춤 영상  □ 팀/그룹 영상  □ 시설 홍보 영상  □ 직접 입력

→ WHEN (언제):
  □ 즉시  □ 3개월 달성 시점  □ 매 분기말  □ 직접 입력

→ WHERE (어디서):
  □ 카카오톡  □ 앱 푸시  □ 이메일  □ 직접 입력

→ HOW (어떻게):
  □ AI 자동 생성  □ 템플릿 적용  □ 수동 편집  □ 직접 입력

→ WHY (왜):
  □ 재등록 유도  □ 후기 유도  □ 바이럴  □ 직접 입력
```

## 📊 PHASE 1: SENSE (감지) - 미래 결과값 예측

### 핵심: 외부 환경 → 미래 결과값 변화량 예측

**결과값 = f(시간, 외부요인)**

### 1-1. 기울기 기반 예측 모델

```
현재 매출: ₩10,000,000/월
현재 기울기: +5%/월

외부 요인 영향:
┌─────────────┬────────────┬─────────────┬──────────────┐
│ 요인        │ 현재 상태   │ 기울기 변화  │ 3개월 후 예측 │
├─────────────┼────────────┼─────────────┼──────────────┤
│ 경쟁 환경   │ 신규 진입   │ -2%        │ 영향: 부정적  │
│ 경제 상황   │ 금리 인상   │ -1%        │ 영향: 부정적  │
│ 기술 트렌드 │ AI 도구    │ +3%        │ 영향: 긍정적  │
│ 계절 요인  │ 여름 시즌   │ +4%        │ 영향: 긍정적  │
│ 고객 행동  │ 온라인 선호 │ +2%        │ 영향: 긍정적  │
└─────────────┴────────────┴─────────────┴──────────────┘

예측 결과 기울기: +5% + (-2%) + (-1%) + 3% + 4% + 2% = +11%/월
3개월 후 예측 매출: ₩10,000,000 × (1.11)³ = ₩13,676,000
```

### 1-2. 시스템이 먼저 제시

```
📊 외부 환경 분석 결과 (자동 생성)

현재 상황:
• 경쟁: 신규 업체 2곳 진입 → 예상 영향 -2%
• 계절: 여름 시즌 진입 → 예상 영향 +4%
• 트렌드: 건강 관심 증가 → 예상 영향 +3%

💡 시스템 추천:
"현재 외부 환경은 긍정적입니다.
여름 시즌을 활용한 프로모션을 권장합니다."

사용자 의견: [___________________]
```

## 📋 9단계 상세 정의

### PHASE 1: SENSE (감지)
- **목적**: 외부 환경 분석 → 미래 결과값 예측
- **프레임워크**: Ray Dalio의 약한 신호 포착
- **출력**: 기울기 변화량 기반 예측치

### PHASE 2: ANALYZE (분석)
- **목적**: 이벤트의 근본 원리 분해
- **프레임워크**: Musk의 제1원리 사고
- **질문**: "물리법칙 수준에서 이것이 가능한가?"
- **시스템 추천**: 근본 원인 분석 결과

### PHASE 3: STRATEGIZE (전략)
- **목적**: 독점/차별화 가능성 검토
- **프레임워크**: Thiel의 Zero to One
- **질문**: "이것으로 작은 시장 100%를 가질 수 있나?"
- **시스템 추천**: 경쟁 분석 및 포지셔닝 제안

### PHASE 4: DESIGN (설계)
- **목적**: 고객 관점에서 역설계
- **프레임워크**: Bezos의 Working Backwards
- **질문**: "출시 보도자료를 먼저 쓴다면?"
- **시스템 추천**: PR/FAQ 초안 자동 생성

### PHASE 5: BUILD (구축)
- **목적**: 실행 조직/자원 설계
- **프레임워크**: Two-Pizza Team
- **질문**: "피자 2판으로 충분한 팀인가?"
- **시스템 추천**: 최소 필요 자원 산정

### PHASE 6: LAUNCH (출시)
- **목적**: MVP 빠른 출시
- **프레임워크**: Reid Hoffman의 MVP Rule
- **질문**: "부끄럽지 않으면 너무 늦은 것"
- **시스템 추천**: MVP 범위 제안

### PHASE 7: MEASURE (측정)
- **목적**: 핵심 지표 설정
- **프레임워크**: North Star Metric
- **질문**: "하나의 핵심 지표는 무엇인가?"
- **시스템 추천**: Input/Output 지표 매핑

### PHASE 8: LEARN (학습)
- **목적**: 피드백 반영
- **프레임워크**: Blameless Post-Mortem
- **질문**: "시스템이 왜 이것을 허용했나?"
- **시스템 추천**: 개선점 자동 도출

### PHASE 9: SCALE (확장)
- **목적**: 성장 구조 설계
- **프레임워크**: Flywheel Effect
- **질문**: "플라이휠이 돌아가는가?"
- **시스템 추천**: 확장 시나리오 제시

## 📐 미래 결과값 예측 공식

### 기본 공식
```
R(t+n) = R(t) × (1 + Σ(기울기_i))^n

R(t): 현재 결과값
R(t+n): n기간 후 예측 결과값
기울기_i: 각 외부요인이 미치는 영향
```

### 8대 외부요인 기울기 매핑
| 요인 | 측정 지표 | 기울기 범위 |
|-----|---------|-----------|
| 경쟁 | 신규 진입자 수, 가격 변화 | -5% ~ +2% |
| 경제 | 금리, 소비심리지수 | -3% ~ +3% |
| 기술 | 신기술 도입 가능성 | -1% ~ +5% |
| 사회 | 트렌드 부합도 | -2% ~ +4% |
| 정책 | 규제 변화 | -4% ~ +2% |
| 계절 | 시즌 영향도 | -3% ~ +5% |
| 트렌드 | 검색량 변화 | -2% ~ +4% |
| 고객 | 행동 변화 | -2% ~ +3% |
