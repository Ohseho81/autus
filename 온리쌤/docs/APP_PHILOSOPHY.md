# 🏀 온리쌤 앱 철학

## 핵심 원칙

> **"앱은 단순 업무만 하게 유도한다. 전략과 운영 효율화는 아키텍처에 녹아있다."**

---

## 역할별 가이드

### 👨‍🏫 코치 (Staff)
**할 수 있는 것** (단순 업무)
- 출석번호 확인 (학생이 입력, 코치는 지켜보기만)
- 영상 업로드 (파일 선택만)
- 결석 예측 알림 확인

**할 수 없는 것** (시스템이 자동)
- 세션 차감 계산
- 학부모 알림 발송
- 보강권 발급
- 미납자 관리

```
[코치 화면 = 3버튼]
┌─────────────────────────────┐
│  📱 출석체크    📹 영상    ⚠️ 알림  │
└─────────────────────────────┘
```

---

### 👩‍💼 관리자 (Admin)
**할 수 있는 것** (단순 업무)
- 대시보드 확인 (읽기만)
- 스케줄 배정 (드래그 앤 드롭)
- 신규 등록 접수
- 상담 기록

**할 수 없는 것** (시스템이 자동)
- 코트 충돌 검사
- 코치 중복 배정 방지
- 미납 알림 발송
- 재고 관리

```
[관리자 = 보기 + 배정 + 등록]
시스템이 이상 감지 → 관리자에게 알림
관리자가 판단 → 클릭 한 번으로 처리
```

---

### 👨‍👩‍👧 학부모 (Consumer)
**할 수 있는 것** (원터치)
- 결석 신고 → 버튼 1번
- 보강 신청 → 버튼 1번
- 스케줄 변경 → 버튼 1번
- 결제 → 버튼 1번

**할 수 없는 것** (시스템이 자동)
- 보강 가능일 검색
- 잔여 회차 계산
- 환불 금액 계산
- 노쇼 패널티 적용

```
[학부모 = 문의 버튼 없음]
모든 것을 셀프서비스로
문의할 일이 없게 만든다
```

## 아키텍처에 녹인 전략

### 1. 자동화 99%
| 업무 | 사람 | 시스템 |
|------|------|--------|
| 출석 | 번호 확인 | 세션 차감, 알림, 기록 |
| 결석 | 사유 선택 | 보강권 발급, 통계 |
| 결제 | 금액 확인 | 할인 계산, 플라이휠 |
| 예약 | 시간 선택 | 재고, 충돌, 대기열 |

### 2. 에스컬레이션 자동화
```
문제 발생
    ↓
[Level 1] 시스템 자동 처리 (0분)
    ↓ 실패
[Level 2] 코치 알림 (즉시)
    ↓ 미처리
[Level 3] 관리자 알림 (4시간)
    ↓ 미처리
[Level 4] 오너 알림 (24시간)
```

### 3. SLA 기반 운영
모든 요청에 응답 시간 목표:
- 출석 체크: 500ms
- 학부모 요청: 즉시~4시간
- 환불 처리: 24시간

**목표 미달성 시 자동 에스컬레이션**

### 4. 플라이휠 효과 (결제)
```
결제 → 출석 → 만족 → 재등록 → 결제
  ↑                              ↓
  └──────── 자동 갱신 ←──────────┘
```
- 첫 결제: 보너스 1회
- 재등록: 5% 할인 + 보너스 2회
- 자동결제: 10% 할인 + 보너스 3회
- 로열티: 최대 10% 추가 할인

## 화면별 단순화 지침

### Staff 화면
```tsx
// ❌ 복잡
<TextInput placeholder="세션 차감 횟수" />
<TextInput placeholder="알림 메시지" />
<Button>저장</Button>

// ✅ 단순
<NumberPad onSubmit={checkAttendance} />
// 끝. 나머지는 시스템이 알아서
```

### Admin 화면
```tsx
// ❌ 복잡
세부 설정 10개
옵션 체크박스 20개

// ✅ 단순
대시보드 (읽기)
+ 빨간불 들어오면 클릭 한 번으로 처리
```

### Consumer 화면
```tsx
// ❌ 복잡
문의하기 버튼
FAQ 100개
상담 예약

// ✅ 단순
[결석] [보강] [스케줄] [결제]
4개 버튼이 전부
문의할 일이 없게 만든다
```

## 오너의 역할

앱 사용 ❌
전략 수립 ✅

오너가 할 일:
1. 가격 정책 결정 → 시스템에 반영
2. 프로모션 기획 → 시스템이 자동 적용
3. 코치 채용/평가 → 데이터 기반 (앱이 제공)
4. 확장 결정 → 대시보드 분석

**오너는 앱을 안 써도 된다. 앱이 오너에게 필요한 것만 알려준다.**
