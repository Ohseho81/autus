## 👥 사용자 정의

### 1. 강사 (Coach) - **최우선**

**페르소나**
- 20~30대, 체육 전공
- 하루 3~5개 수업
- 스마트폰 능숙하지만 앱 싫어함
- "수업에 집중하고 싶다"

**핵심 니즈**
- 빠르게 출석 처리
- 행정 업무 최소화
- 사고 발생 시 즉시 보고

### 2. 관리자 (Admin) - **두 번째**

**페르소나**
- 30~50대, 학원 원장/실장
- 여러 지점 또는 단일 지점 운영
- 매출/출석/강사 관리 필요
- "숫자로 현황을 보고 싶다"

**핵심 니즈**
- 실시간 운영 현황
- 매출/결제 추적
- 학생 이탈 예측

### 3. 학부모 (Parent) - **Phase 1.5**

**페르소나**
- 30~40대, 자녀 레슨 관리
- 카카오톡 중심 소통
- "우리 아이 잘 하고 있나?"

**핵심 니즈**
- 출석 알림
- 레슨 영상 시청
- 일정 확인

## 📱 앱 스펙

### 강사앱 (v3.0 FREEZE) ✅ 완료

```
┌─────────────────────────────────────┐
│ 🏀 온리쌤              [⚙️ ▼]  │
├─────────────────────────────────────┤
│ 2월 4일 (화) · 대치지점             │
├─────────────────────────────────────┤
│  ┌───────────────────────────────┐  │
│  │ 16:00 유소년 A반      🟢진행중│  │
│  │ 코트 A · 10명 · 경과 18분     │  │
│  │              [수업 종료]      │  │
│  └───────────────────────────────┘  │
├─────────────────────────────────────┤
│         🚨 사고 발생 신고           │
└─────────────────────────────────────┘
```

**제약 조건 (FREEZE)**
| 항목 | 값 | 이유 |
|------|---|------|
| 화면 수 | 1 | 복잡성 제거 |
| 버튼 수 | 3 | 판단 제거 |
| 입력 폼 | 0 | 책임 제거 |
| 네비게이션 | 0 | 이탈 방지 |

**이벤트 (3종만)**
- `SESSION_START` - 수업 시작
- `SESSION_END` - 수업 종료
- `INCIDENT_FLAG` - 사고 신고

### 관리자앱 스펙 (v1.0)

```
┌─────────────────────────────────────┐
│ 📊 온리쌤 관리자                │
├─────────────────────────────────────┤
│                                     │
│   오늘 매출        이번 달 매출     │
│   ₩1,250,000      ₩15,800,000     │
│                                     │
│   진행중 수업      오늘 출석률      │
│      3개            87%            │
│                                     │
├─────────────────────────────────────┤
│ ⚠️ 주의 필요 학생 3명               │
│  • 김민준 - 잔여 2회                │
│  • 이서연 - 결석 2회 연속           │
│  • 박지호 - 결제 예정일 D-3         │
├─────────────────────────────────────┤
│ [대시보드] [학생] [스케줄] [매출] [더보기] │
└─────────────────────────────────────┘
```

**핵심 기능**
| 탭 | 기능 |
|----|------|
| 대시보드 | 오늘 요약, 주의 학생, 알림 |
| 학생 | 학생 목록, 등록, 상세 |
| 스케줄 | 수업 일정, 출석 현황 |
| 매출 | 결제 현황, 매출 추이 |
| 더보기 | 설정, 강사 관리 |

**데이터 모델**
```typescript
interface Student {
  id: string;
  name: string;
  phone: string;
  parent_phone: string;
  remaining_lessons: number;
  skill_level: 'beginner' | 'intermediate' | 'advanced';
  risk_score: number;  // 이탈 위험도 0~100
  created_at: string;
}

interface Payment {
  id: string;
  student_id: string;
  amount: number;
  lessons_purchased: number;
  payment_date: string;
  payment_method: 'card' | 'transfer' | 'cash';
}
```
