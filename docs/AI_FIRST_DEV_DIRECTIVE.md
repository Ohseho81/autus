# 🛠 AI-First 개발 지침 (개발진용)

> 사장님 의지에 따른 공식 지침. AI·LLM·자동화 도구를 **100% 신뢰**하고, 기술의 잠재력을 끝까지 밀어붙여 결과물을 만드는 것을 원칙으로 한다.

---

## 1. AI 도구 활용 극대화 (100% Trust Policy)

### 코드 생성 및 리뷰
- **모든 비즈니스 로직·알고리즘 초안**은 LLM(Claude 3.5 Sonnet, GPT-4o 등)이 작성한 것을 **기본으로 채택**한다.
- 개발자는 AI가 제안한 구조를 의심하기보다, **그 코드를 즉시 실행 가능한 상태로 통합하는 방법**에만 집중한다.

### 디버깅 자동화
- 오류 발생 시, 사람이 직접 디버깅하기 전에 **에러 로그 전체를 LLM에 전달**하고, LLM이 제안한 해결책을 **1순위로 적용**한다.

---

## 2. MCP 및 자동화 툴 설정 강제

### 환경 복구
- MCP 서버 오류 등 **툴 결함은 "안 된다"고 포기할 대상이 아니다.**
- **AI에게 직접 해결 방법을 물어보고**, 설정을 강제 수정하여 **반드시 동작하도록 뚫어낼 관문**으로 규정한다.

### Claude Code 활용
- `claude-code` 등 CLI 도구를 적극 사용하여, **자연어로 코드 수정·파일 구조 변경**이 되는 **무클릭 개발** 환경을 지향한다.

---

## 3. 데이터 중심 의사결정

- **강사·기획자의 주관적 판단보다**, MediaPipe·Whisper 등으로 수집된 **Raw 데이터를 AI가 해석한 결과**를 **100% 신뢰**하여 리포트를 생성한다.

---

## 4. 개발진 전달용 지시 텍스트 (Copy & Paste)

아래 문단을 단톡·이슈·회의록에 그대로 사용할 수 있다.

---

**"우리는 AI와 LLM의 능력을 100% 신뢰하고 개발을 진행합니다."**

- **MCP 서버 오류 즉시 해결:** 현재 발생한 claude-code 연결 문제는 도구의 한계가 아니라 **설정의 문제**입니다. AI를 활용해 즉시 트러블슈팅하고 개발 환경을 완벽하게 세팅하세요.

- **AI 생성 로직 우선:** 6초 영상 클리핑 알고리즘과 AI 리포트 변환 프롬프트는 **LLM이 제안하는 최적의 모델을 그대로 따릅니다.**

- **속도 중심:** 완벽한 코드를 직접 짜려 하지 말고, **AI가 짠 코드를 빠르게 테스트하고 배포하는 '오케스트레이터'** 역할에 집중해 주십시오.

---

## 5. 사용자 권장 사항 (Capability & Correction)

비전을 실현하는 과정에서 AI가 제공하는 정보나 코드가 완벽하지 않을 수 있습니다.

- **직접적인 교정:** 생성된 코드나 비즈니스 로직에 수정이 필요하다고 판단되면, **대화창에서 즉시 직접 교정**해 주십시오.
- AI는 그 피드백을 **즉각 반영**하여 다음 응답에 반영하며 계속 개선됩니다.
