## 📊 5단계 상세

### 1️⃣ DISCOVER (수집 + 노하우 통합)

**질문**: "이 업무가 실제로 존재하는가? K·I 상수는?"

| 항목 | 내용 |
|------|------|
| 데이터 수집 | 업무 메트릭, 빈도, 소요 시간, 관계자 |
| 노하우 통합 | 베스트 프랙티스 패턴 매칭 |
| K·I 초기화 | 카테고리별 템플릿 적용 |
| 존재 검증 | 빈도 0, 시간 0이면 존재 불확실 |

```python
# 예시
work = WorkItem("온보딩 프로세스", category="HR")
→ K=0.6 (HR 템플릿), 자동화 가능성 70% (onboarding 패턴 매칭)
```

---

### 2️⃣ ANALYZE (분석)

**질문**: "이 업무는 번영하는가, 소멸하는가?"

| 지수 | 기준 | 판정 |
|------|------|------|
| **K** ≥ 0.7 | 번영 임계 | 🚀 핵심 업무 (유지) |
| **K** ≤ 0.3 | 쇠퇴 임계 | ⚠️ 삭제 후보 |
| **I** ≤ -0.3 | 부정적 상호작용 | 💥 분리 필요 |
| **Ω** ≤ 0.4 | 저효율 | 🐢 재설계 필요 |

```
종합 점수 = (K + Ω) / 2 - |min(0, I)|

점수 ≥ 0.6 → 번영
점수 ≤ 0.3 → 소멸
그 외 → 유지 (최적화)
```

---

### 3️⃣ REDESIGN (재정의 + 자동화)

**목표**: "최소 에너지 최대 출력 궤적으로 재설계"

| 액션 | 조건 | 효과 |
|------|------|------|
| 🤖 **AUTOMATE** | 자동화 점수 ≥ 80% | 시간 80-95% 절감 |
| 📦 **COMPRESS** | 병합 가능 업무 | 시간 40% 절감 |
| 🧠 **DELEGATE** | AGI 대리인 가능 | 시간 95% 절감 |
| 🗑️ **ELIMINATE** | K < 0.3 | 완전 삭제 |
| 👤 **KEEP** | 핵심 업무 | 프로세스 개선만 |

**자동화 도구 매핑**:
```
data_entry      → RPA Bot (90% 절감)
scheduling      → Calendar AI (85% 절감)
invoice         → OCR + RPA (95% 절감)
report          → BI Dashboard (70% 절감)
email           → AI Assistant (60% 절감)
approval        → Workflow Engine (80% 절감)
onboarding      → HRIS + Chatbot (70% 절감)
```

---

### 4️⃣ OPTIMIZE (피드백 + 최적화)

**목표**: "K·I 재계산 → 효율 상승 확인"

| 피드백 | 측정 | 조정 |
|--------|------|------|
| 실제 시간 | 예상 대비 비율 | Ω ±0.03 |
| 품질 점수 | 0-100% | K ±0.02~0.04 |
| 사용자 피드백 | 1-5점 | K·Ω ±0.02 |

```
예상보다 20% 빠름 → Ω +0.03
품질 90% 이상 → K +0.04
피드백 4점 이상 → K·Ω +0.02
```

**모니터링 주기**: 7일

---

### 5️⃣ ELIMINATE (삭제 + 반복)

**목표**: "자연 소멸 → 빈 슬롯 재수집"

| 삭제 조건 | 기준 |
|----------|------|
| K 임계 | K < 0.3 |
| I 임계 | I < -0.3 |
| 정체 | 30일 변화 없음 + Ω < 0.4 |
| 강제 | 사용자 요청 |

```
삭제 시:
- 월간 소요 시간 전체 회수
- 에너지 100% 절감
- 빈 슬롯 → 새 업무 수집 대기
```

## 💻 사용 예시

```python
from core.unified.solution_loop import SolutionLoop, WorkItem

# 루프 엔진 초기화
loop = SolutionLoop()

# 업무 추가
work = WorkItem(
    id="w1",
    name="월간 청구서 처리",
    category="Finance",
    frequency=30,           # 월 30회
    duration_hours=1.5,     # 건당 1.5시간
    complexity=0.3          # 낮은 복잡도
)
loop.add_work(work)

# 전체 루프 실행
results = loop.run_full_loop("w1")

# 결과 확인
for result in results:
    print(f"{result.phase}: {result.action}")
    # 1. DISCOVER: KEEP
    # 2. ANALYZE: AUTOMATE (자동화 권장)
    # 3. REDESIGN: AUTOMATE (OCR+RPA 적용)
    # 4. OPTIMIZE: KEEP (모니터링)
    # 5. ELIMINATE: KEEP (유지)

# 대시보드
print(loop.generate_dashboard())
```

## 🎯 핵심 원칙

1. **데이터 발견**: 업무가 실제로 존재하는지 검증

3. **최소 개입**: 시스템은 제안만, 결정은 인간이
4. **에너지 효율**: 최소 에너지 최대 출력 궤적
