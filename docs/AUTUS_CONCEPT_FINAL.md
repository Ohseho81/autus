## 3) 결과 분류 (What Happens After Choice)

모든 선택의 결과는 **4가지 중 하나**로 귀결된다:

| # | 결과 | 영문 | 설명 |
|---|------|------|------|
| 1 | 가치 창출 | Create | 새로운 가치 생성 |
| 2 | 가치 유지 | Preserve | 기존 가치 보존 |
| 3 | 가치 소모 | Consume | 가치의 계획적 사용 |
| 4 | 가치 소멸 | Destroy | 복구 불가능한 손실 |

> AUTUS는 "무엇을 하라"가 아니라 **어디로 가는지**만 보여준다.

## 6) 엔진 최소 수식 (How It Computes)

### 핵심 변수

| 변수 | 정의 | 공식 |
|------|------|------|
| **TC** | Total Cost (누적 기회비용) | Σ(Cost_i × Weight_i × TimeDecay_i) |
| **CV** | Cost Velocity (비용 증가 속도) | d(TC)/dt |
| **PNR_ETA** | 불가역까지 남은 시간 | (PNR_TC − TC) / CV |
| **VB** | Value Battery (남은 에너지) | 1 − TC/PNR_TC |

### 상태 머신

```
SAFE → WARNING → CRITICAL → IRREVERSIBLE
```

| 상태 | PNR (일) | ACTION |
|------|----------|--------|
| SAFE | > 21 | 표시 |
| WARNING | 7-21 | 표시 |
| CRITICAL | 1-7 | 표시 + 강조 |
| IRREVERSIBLE | 0 | 제거 |

### 금지 규칙

```
❌ 예측
❌ 추천
❌ 비교
⭕ 현재 선택 유지의 비용만 계산
```

## 8) 도착 이후의 핵심 (Why It Compounds)

### 가치 축적 순환

```
결정 완료 → Audit 고정 → 과정 데이터 누적 → 표준 정밀화 → 다음 결정
```

### 결정 가속 공식

```
Velocity(n+1) = Velocity(n) × Standard_Precision
```

### 결정이 쌓일수록:

| 지표 | 변화 |
|------|------|
| 속도 | ↑ 더 빠름 |
| 비용 | ↓ 더 싸짐 |
| 명확성 | ↑ 더 명확함 |

> **결정이 쌓일수록 가속도가 증가한다.**
