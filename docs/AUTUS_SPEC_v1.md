## 제2공리: 시간의 상대성

```
┌─────────────────────────────────────────────┐
│                                             │
│   동일한 시간도 노드마다 가치가 다르다       │
│                                             │
│   t_표준 = t_실제 × λ                       │
│                                             │
└─────────────────────────────────────────────┘

λ = 노드 고유 시간상수
→ 원장의 1시간 ≠ 신입의 1시간
→ 각 노드의 λ로 표준시간 환산
```

## 제3공리: 시너지의 지수성

```
┌─────────────────────────────────────────────┐
│                                             │
│   관계의 시너지는 시간에 지수로 작용한다     │
│                                             │
│   V ∝ e^(σt)                               │
│                                             │
└─────────────────────────────────────────────┘

σ > 0: 시너지 (함께하면 기하급수 성장)
σ = 0: 중립 (변화 없음)
σ < 0: 역시너지 (함께하면 기하급수 소멸)
```

## 3.2 변수 정의

### λ (Lambda): 노드 시간상수

```
┌─────────────────────────────────────────────┐
│                                             │
│   λ = 노드의 1시간이 갖는 표준시간 가치      │
│                                             │
│   "이 사람의 1시간은 몇 시간의 가치인가"     │
│                                             │
└─────────────────────────────────────────────┘

계산 공식:
λ = λ_base × (1/R) × I × E × N

R = Replaceability (대체 가능성, 0~1)
I = Influence (영향력, 0~1)
E = Expertise (전문성, 0~1)
N = Network Position (네트워크 위치, 0~1)
λ_base = 1.0 (기준)

제약:
λ_min = 0.5
λ_max = 10.0
```

**역할별 기본값 (v1.0):**

| 역할 | λ | 근거 |
|------|---|------|
| 원장/대표 | 5.0 | 대체 불가, 최대 영향력 |
| 실장/부원장 | 3.5 | 낮은 대체성, 높은 영향력 |
| 수석 강사 | 3.0 | 전문성 높음 |
| 일반 강사 | 2.0 | 중간 전문성 |
| 신입 강사 | 1.5 | 학습 중 |
| 행정 직원 | 1.5 | 대체 가능 |
| 학생 | 1.0 | 기준 노드 |
| 학부모 | 1.2 | 의사결정권 |

---

### σ (Sigma): 시너지 계수

```
┌─────────────────────────────────────────────┐
│                                             │
│   σ = 두 노드 간의 시너지 계수              │
│                                             │
│   "둘이 만나면 시간이 늘어나는가 줄어드는가" │
│                                             │
└─────────────────────────────────────────────┘

계산 공식:
σ = w₁C + w₂G + w₃V + w₄R

C = Compatibility (스타일 호환, -1~+1)
G = Goal Alignment (목표 일치, -1~+1)
V = Value Match (가치관 일치, -1~+1)
R = Rhythm Sync (리듬 동기화, -1~+1)

가중치 (v1.0):
w₁ = 0.3, w₂ = 0.3, w₃ = 0.2, w₄ = 0.2

범위: -1.0 ≤ σ ≤ +1.0
```

**시너지 효과:**

| σ | t=12개월 | 해석 |
|---|----------|------|
| +0.3 | ×36.6배 | 탁월한 매칭 |
| +0.1 | ×3.3배 | 좋은 매칭 |
| 0 | ×1배 | 중립 |
| -0.1 | ×0.3배 | 나쁜 매칭 |
| -0.2 | ×0.09배 | 독성 관계 |

---

### P (Density): 관계 밀도

```
┌─────────────────────────────────────────────┐
│                                             │
│   P = 관계의 실질적 연결 강도               │
│                                             │
│   "이 관계는 얼마나 실제로 작동하는가"       │
│                                             │
└─────────────────────────────────────────────┘

계산 공식:
P = √(F × D) × Q_adj

F = Frequency (접촉 빈도, 0~1)
D = Depth (관계 깊이, 0~1)
Q_adj = Quality 보정 (0~1)

범위: 0 ≤ P ≤ 1
```

**P 측정 기준:**

| 빈도 (F) | 값 |
|----------|-----|
| 매일 | 1.0 |
| 주 2-3회 | 0.8 |
| 주 1회 | 0.6 |
| 월 2회 | 0.4 |
| 월 1회 | 0.2 |

| 깊이 (D) | 값 |
|----------|-----|
| 파트너 | 1.0 |
| 의존 | 0.8 |
| 신뢰 | 0.6 |
| 친숙 | 0.4 |
| 인지 | 0.2 |

### T (Time): 시간 체계

```
시간의 3가지 유형:

T₁ (투입) = 관계 유지에 쓴 시간 (Cost)
T₂ (절약) = 관계로 줄어든 시간 (Save)
T₃ (창출) = 관계로 생긴 미래 시간 (Create)

순시간가치:
NTV = T₃ - T₁ + T₂
    = 창출 - 투입 + 절약
```

## 3.3 단위 체계

```
┌─────────────────────────────────────────────┐
│                                             │
│   STU (Standard Time Unit) = 표준시간       │
│                                             │
│   1 STU = λ=1.0인 노드의 1시간              │
│                                             │
└─────────────────────────────────────────────┘

변환:
실제 시간 → 표준 시간: t_STU = t_real × λ
표준 시간 → 화폐 가치: V_₩ = t_STU × ω

ω = 시간 단가 (₩/STU)
  = 총 매출 / 총 투입 STU
```

## 4.2 데이터 수집 방법

### T₁ (투입 시간) 측정

| 활동 | 측정 방법 | 자동화 |
|------|----------|--------|
| 수업 시간 | 출석 시스템 연동 | ✅ 자동 |
| 상담 시간 | 캘린더 연동 | ✅ 자동 |
| 통화 시간 | 통화 로그 | ✅ 자동 |
| 메시지 시간 | 작성 시간 추정 | ⚠️ AI 추정 |
| 준비 시간 | 수동 입력 | ❌ 수동 |

### T₂ (절약 시간) 측정

| 자동화 항목 | Before | After | 절약 |
|------------|--------|-------|------|
| 자동 알림 | 10시간/월 | 1시간 | 9시간 |
| AI 리포트 | 8시간/월 | 1시간 | 7시간 |
| 자동 결제 | 5시간/월 | 0.5시간 | 4.5시간 |
| 매칭 시스템 | 4시간/월 | 0.5시간 | 3.5시간 |

### T₃ (창출 시간) 측정

```
T₃ = Σ(예상 시간_i × 확률_i)

구성요소:
- 예상 잔존 기간 × 잔존 확률
- 예상 추천 수 × 추천 확률  
- 예상 업그레이드 × 전환 확률

AI 예측 모델:
R(t) = Σ(wᵢ × ΔMᵢ) / s(t)^α
→ 이탈 확률 예측
→ T₃ 계산
```

## 5.2 단계별 적용 로드맵

### Phase 1: 기본 측정 (Month 1)

```
□ 역할별 λ 고정값 설정
□ T₁ 자동 기록 시스템 구축
  ├─ 수업 시간 (출석 연동)
  ├─ 상담 시간 (캘린더 연동)
  └─ 메시지 시간 (추정)
□ P 자동 계산
  ├─ F = 월 접촉 횟수 / 기대 횟수
  └─ D = Quick Tag 기반 단계
□ V_simple 주간 리포트
```

### Phase 2: 파생 계산 (Month 2-3)

```
□ T₂ 측정 (자동화 전후 비교)
□ T₃ 기초 예측 (잔존 기간 추정)
□ NTV 계산 (T₃ - T₁ + T₂)
□ λ 분기 조정 프로세스
□ 대시보드 구축
```

### Phase 3: 시너지 도입 (Month 4-6)

```
□ σ 사후 측정 시작
□ 매칭 결과 데이터 축적
□ σ 예측 모델 학습
□ e^(σt) 적용 시작
□ 풀버전 V = P × Λ × e^(σt)
```

## 6.3 보정 대상별 정교화

### λ 정교화

```
v1.0: 역할 기반 고정값
      ↓
v1.1: 분기별 수동 조정 (±20%)
      ↓
v1.2: 성과 연동 반자동 조정
      ↓
v2.0: AI 기반 실시간 동적 λ

보정 공식:
λ_new = λ_old × (1 + α × Δperformance)
α = 학습률 (기본 0.1)
```

### σ 정교화

```
v1.0: 사후 측정 (3개월 후 평가)
      ↓
v1.1: 유사 케이스 기반 추정
      ↓
v1.2: 회귀 모델 예측
      ↓
v2.0: 딥러닝 예측 + 실시간 조정

보정 공식:
σ_pred = f(C, G, V, R) + ε
ε = 오차 보정 (실측값 피드백)
```

### P 정교화

```
v1.0: F × D (단순 곱)
      ↓
v1.1: √(F × D) × Q_adj
      ↓
v1.2: 가중치 최적화
      ↓
v2.0: 동적 가중치

보정 공식:
P = w₁F + w₂D + w₃Q
w_i = 데이터 기반 최적화
```

## 6.4 피드백 루프 설계

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│                  피드백 루프                        │
│                                                     │
│   ┌─────────┐     ┌─────────┐     ┌─────────┐     │
│   │  측정   │────→│  예측   │────→│  실행   │     │
│   │ Measure │     │ Predict │     │  Act    │     │
│   └─────────┘     └─────────┘     └─────────┘     │
│        ↑                                │          │
│        │          ┌─────────┐          │          │
│        └──────────│  학습   │←─────────┘          │
│                   │  Learn  │                      │
│                   └─────────┘                      │
│                                                     │
└─────────────────────────────────────────────────────┘

1. 측정 (Measure)
   - 상호작용 로그 수집
   - T₁, P 자동 계산
   - Quick Tag 입력

2. 예측 (Predict)
   - V 계산
   - T₃ 예측
   - σ 추정

3. 실행 (Act)
   - 위험 알림
   - 매칭 추천
   - 자동 개입

4. 학습 (Learn)
   - 예측 vs 실측 비교
   - 오차 분석
   - 파라미터 보정
```

```
                  │
       ┌──────────┼──────────┐

       │  보정   │  시작    │
       ├──────────┼──────────┤
       │    P     │   T₃     │
 어려움│  가중치  │  예측    │

       └──────────┼──────────┘
                  │

순서:

3. P 가중치 조정 (어려움, 영향도 낮음)

```
