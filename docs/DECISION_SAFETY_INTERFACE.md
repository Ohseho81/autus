# 🏛️ AUTUS: Decision Safety Interface

## 선언문 (Declaration)

---

> **"이 인터페이스는 사용자를 편하게 만들기 위해 설계되지 않았다.**
> **틀린 결정을 하기 어렵게 만들기 위해 설계되었다."**

---

## 1. 철학적 기반

### 1.1 기존 UI의 문제

| 기존 UI 설계 | 결과 |
|-------------|------|
| "모든 정보를 보여준다" | 인지 과부하 → 잘못된 판단 |
| "빠르게 실행한다" | 비가역적 실수 → 복구 불가 |
| "모든 사용자에게 동일하다" | 권한 초과 → 책임 불명확 |
| "되돌리기 쉽다" | 비가역성 망각 → 중대 결정 경시 |

### 1.2 AUTUS의 해결책

| AUTUS 원칙 | 구현 |
|-----------|------|
| **판단 정확도 > 편의성** | 결정 고도(K)에 따른 UI 제한 |
| **비가역성 인지** | 색온도, 드래그 저항, Ritual Gate |
| **권한 가시성** | Fog, Blackhole, 차등 해상도 |
| **책임 명확화** | 승인 주체 표시, Override 로그 |

## 2. K-Scale 시스템

### 2.1 10단계 의사결정 고도

```
K1  ────  개인 실행      (분~시간)     개인
K2  ────  현장 운영      (시간)       현장 책임자
K3  ────  팀/부서 운영   (일)         중간 관리자
K4  ────  조직 설계      (주)         경영진
K5  ────  사업 선택      (월)         오너/이사회
K6  ────  제도·규제      (분기~년)    법적 승인
K7  ────  블록 정렬      (수년)       다자 합의
K8  ────  문명 영향      (세대)       사회적 합의
K9  ────  자본 질서      (세대+)      초국가 자본
K10 ────  헌법/원칙      (문명 단위)  창시자/헌법
```

### 2.2 자동 판별 기준

각 K단계는 두 가지로 명확히 구분된다:

1. **누가 승인하는가** (Approval Authority)
2. **실패 시 무엇이 깨지는가** (Failure Cost)

## 3. 6대 완성도 평가 축

AUTUS UI는 일반 UX 척도가 아닌, 다음 6가지 축으로 평가된다:

### ① Decision Fidelity (판단 정확도) ★★★★★
> UI가 틀린 결정을 얼마나 줄이는가

- Gravity Trigger
- Altitude Lock
- Ω 기반 강제 전이

### ② Cognitive Load Control (인지 부하 제어) ★★★★★
> 지금 필요한 정보만 보이게 하는가

- K별 UI 차단
- Observer Altitude
- Fog / Blur / Lensing

### ③ Irreversibility Awareness (비가역성 인지) ★★★★★
> 사용자가 "되돌릴 수 없음"을 체감하는가

- 색온도 변화 (따뜻함 → 차가움)
- Drag 증가
- Ritual 진입

### ④ Authority Legibility (권한 가시성) ★★★★★
> 누가 어디까지 결정 가능한지가 직관적인가

- Blackhole / Fog
- 시각적 위계
- 차등 해상도

### ⑤ Scalability of Meaning (의미 확장성) ★★★★★
> 개인 → 조직 → 문명까지 동일한 언어로 작동하는가

- K1~K10 프랙털
- 동일 판단 공식 (K·I·Ω·r)
- 동일 UI 철학

### ⑥ Ethical Boundary Clarity (윤리적 경계) ★★★★☆
> 시스템이 어디까지 개입하고 어디서 멈추는가

- "최종 책임은 인간에게 있습니다"
- Override 로그
- 최종 승인 고정

## 4. 핵심 UI 메커니즘

### 4.1 Gravity Trigger

결정의 중력에 따라 UI가 자동으로 반응한다.

| 조건 | 최소 고도 |
|------|----------|
| 개인 작업 | K1 |
| 팀 리소스 변경 | K3 |
| 조직 구조 변경 | K4 |
| 신규 사업/투자 | K5 |
| 계약·세무·규제 | **K6 강제** |
| 다국가/다시장 | K7 |
| ESG·인류 영향 | K8 |
| 메가펀드/금융질서 | K9 |
| 시스템 헌법 수정 | **K10 단독** |

### 4.2 Ritual Gate

K4 이상의 결정은 **의식적 진입**이 필요하다.

```
[책임 확인] → [비가역성 인지] → [최종 확인 입력]
```

- 단순 클릭으로는 진입 불가
- 텍스트 입력 또는 드래그로 의도 확인
- 각 단계에서 취소 가능

### 4.3 색온도 시스템

| K단계 | 색온도 | 느낌 |
|-------|--------|------|
| K1~K3 | 5500~6200K | 중립~약간 차가움 |
| K4~K5 | 4500~6800K | 따뜻함 (주의) |
| K6 | 3500K | 붉은 경고 |
| K7~K8 | 7500~8500K | 차가움 (신중) |
| K9 | 3000K | 금색 (자본) |
| K10 | 10000K | 순백 (신성) |

## 5. 대상 사용자

### 5.1 이 UI는 누구를 위한 것인가

✅ **적합**
- CEO / C-Level 경영진
- 정책 결정자
- 인프라 운영자
- 투자 의사결정자
- 시스템 아키텍트

❌ **부적합**
- 일반 소비자
- 단순 작업자
- 빠른 실행이 중요한 업무

### 5.2 왜 모든 사용자용이 아닌가

> "실수해도 되는 결정"과 "실수하면 안 되는 결정"은
> 동일한 인터페이스로 처리해서는 안 된다.

AUTUS는 후자를 위해 설계되었다.

## 6. 결론

### 한 문장 요약

> **AUTUS UI는 "편한 UI"가 아니라 "틀리기 어려운 UI"다.**

### 핵심 가치

1. **판단 정확도**: 잘못된 결정 방지
2. **비가역성 인지**: 되돌릴 수 없음을 체감
3. **책임 명확화**: 누가 승인하는지 명시
4. **스케일 확장성**: 개인부터 문명까지 동일 언어
