## 1.2 핵심 개념: A = T^σ

AUTUS의 모든 기능은 하나의 공식에서 시작합니다:

```
A = T^σ

- A (Asset): 자산 가치 (학생의 LTV)
- T (Transaction): 거래/시간
- σ (Sigma): 만족도 지수 (0~2)
```

**σ > 1**: 자산이 성장 (유지/추천 예상)  
**σ < 1**: 자산이 감소 (이탈 위험)  
**σ = 1**: 현상 유지

# 3. 역할별 가이드

## 3.1 👑 결정자 (C-Level / Tier 1)

> **철학**: "결정만 한다. 과정·설계·자동화는 보이지 않는다."

### 접근 가능한 기능

| 탭 | 기능 | 설명 |
|---|------|------|
| **👑 Monopoly** | 독점 대시보드 | 데이터·가치·네트워크 3대 독점 현황 |
| **⚖️ 결정** | 의사결정 | 승인/보류/거부 결정 |
| **📊 자산화** | 자산 현황 | 570개 업무 자산화 지표 |
| **🔮 시나리오** | 미래 예측 | AI 기반 시나리오 분석 |

### 핵심 지표

```
V-Index = (M - T) × (1 + s)^t
├─ M: 매출 (Mint)
├─ T: 비용 (Tax)
├─ s: 만족도 지수
└─ t: 시간 (개월)
```

### 사용 방법

1. **Monopoly 탭** 선택
2. 3대 독점 (데이터/가치/네트워크) 현황 확인
3. 목표 현황 및 이탈 위험 요약 확인
4. 필요시 **결정 탭**에서 승인/거부 처리

## 3.2 ⚙️ 운영자 (FSD / Tier 2)

> **철학**: "관리의 기준을 설명에서 증거로 바꾼다."

### 접근 가능한 기능

| 탭 | 기능 | 설명 |
|---|------|------|
| **🚨 Risk Queue** | 위험 관리 | 이탈 위험 학생 목록 및 조치 |
| **⚠️ 이탈 알림** | 이탈 분석 | 이탈 예측 및 방어 전략 |
| **⚡ 충돌** | 충돌 감지 | 업무 충돌 감지 및 해결 |
| **🔥 압력맵** | 압력 분석 | 영역별 압력 히트맵 |
| **📋 업무재정의** | 업무 최적화 | 570개 업무 통합/삭제/자동화 |

### 핵심 지표

```
R(t) = Σ(wᵢ × ΔMᵢ) / s(t)^α
├─ wᵢ: 시간 가중치 (최근 데이터 우선)
├─ ΔMᵢ: 성과 변화량
├─ s(t): 현재 만족도
└─ α: 만족도 민감도 (1.5)
```

### Risk Queue 사용 방법

1. **🚨 Risk Queue 탭** 선택
2. 우선순위별 위험 목록 확인 (Critical → High → Medium → Low)
3. 위험 항목 클릭하여 상세 정보 확인
4. 조치 선택:
   - **🛡️ Active Shield**: 자동 방어 발동 (긍정 리포트 발송 + 담당자 알림)
   - **✓ 해결**: 위험 해결 처리
   - **⬆️ 에스컬레이션**: 상위 결정자에게 전달
   - **✕**: 오탐 처리 (dismiss)

### 이탈 알림 사용 방법

1. **⚠️ 이탈 알림 탭** 선택
2. 위험 분포 확인 (긴급/높음/보통/낮음)
3. 긴급 대응 필요 목록 확인
4. **🛡️ 방어** 버튼으로 Active Shield 발동

## 3.3 🔨 실행자 (Optimus / Tier 3)

> **철학**: "생각하지 않게 한다. 다음 행동만 보여준다."

### 접근 가능한 기능

| 탭 | 기능 | 설명 |
|---|------|------|
| **⚡ Quick Tag** | 빠른 태깅 | 현장 데이터 즉시 입력 |
| **📋 작업** | 다음 작업 | 현재 수행할 작업 표시 |
| **📄 보고서** | 자동 보고서 | AI 자동 생성 보고서 |

### 핵심 지표

```
Execution Score = (완료 태스크 / 전체 태스크) × 자동화율
```

### Quick Tag 사용 방법

1. **⚡ Quick Tag 탭** 선택
2. **대상 선택**: 학생/학부모/선생님 ID 입력
3. **감정 변화** 입력: 😊매우좋음 ~ 😠나쁨 (또는 슬라이더로 -20 ~ +20)
4. **유대 관계** 선택: 💪강함 / 😐보통 / ❄️냉담
5. **이슈 태그** 선택: 성적, 출결, 태도, 비용 등
6. (선택) **음성 메모** 입력: AI가 분석하여 위험 신호 감지
7. **⚡ Quick Tag 등록** 버튼 클릭

### Quick Tag 자동 처리

- 감정 변화 ≤ -15 또는 유대관계 = 냉담 → **자동으로 Risk Queue에 추가**
- AI가 위험 신호 감지 → **n8n 웹훅으로 Active Shield 발동**

# 4. 핵심 기능

## 4.1 Risk Queue (이탈 위험 관리)

### 우선순위 레벨

| 레벨 | 색상 | 기준 | 권장 조치 |
|-----|------|------|----------|
| **CRITICAL** | 🔴 빨강 | 점수 ≥ 80 | 즉시 1:1 상담, 원장 직접 연락 |
| **HIGH** | 🟠 주황 | 점수 60~79 | 담당 선생님 특별 케어 |
| **MEDIUM** | 🟡 노랑 | 점수 40~59 | 학부모 앱 푸시 알림 |
| **LOW** | 🔵 파랑 | 점수 < 40 | 모니터링 |

### 위험 점수 계산

```
R(t) = -weightedSum / s(t)^α

weightedSum = Σ(timeWeight × categoryWeight × emotionDelta)
├─ timeWeight = e^(-daysSince/30)  // 30일 반감기
├─ categoryWeight: 결제(1.5), 출결(1.2), 성적(1.0), 참여(0.8)
└─ emotionDelta: Quick Tag에서 입력된 감정 변화

riskScore = 50 + R(t) × 10
```

## 4.2 Active Shield (선제적 방어)

Risk Queue에서 **🛡️ Active Shield** 버튼을 클릭하면:

1. **긍정 리포트 자동 발송**: 해당 학생의 긍정적 성과 리포트를 학부모에게 발송
2. **담당자 알림**: 담당 선생님에게 특별 케어 요청 알림
3. **n8n 워크플로우**: 추가 자동화 액션 실행

## 4.3 Quick Tag (현장 데이터 입력)

### 입력 항목

| 항목 | 설명 | 범위 |
|-----|------|------|
| 대상 ID | 학생/학부모/선생님 ID | 텍스트 |
| 감정 변화 | 상호작용 후 감정 변화 | -20 ~ +20 |
| 유대 관계 | 현재 관계 상태 | 강함/보통/냉담 |
| 이슈 태그 | 주요 이슈 분류 | 성적, 출결, 태도, 비용 등 |
| 음성 메모 | AI 분석용 텍스트 | 자유 입력 |

### 자동 처리

- **s-index 업데이트**: 학생의 만족도 지수 자동 계산
- **Physics Metrics 업데이트**: 물리 엔진 메트릭 갱신
- **Immortal Ledger 기록**: 불변 감사 로그 저장
- **위험 감지**: 조건 충족 시 Risk Queue 자동 추가

# 7. 동기부여 설계

AUTUS는 **"생물학적 보상(도파민)으로 즉각 동기를, 인문학적 의미로 지속 동기를 만든다"** 원칙을 따릅니다.

## 7.1 역할별 동기부여 요약

| 역할 | 도파민 트리거 | 의미 부여 | UI 핵심 |
|-----|-------------|----------|---------|
| 🔨 **선생님** | 완료 체크, 내 행동→결과, 연속 기록 | "내가 이 아이를 변화시켰다" | 진행률 바, 🔥연속기록 |
| ⚙️ **실장님** | 숫자 개선, 문제 해결, 예측 적중 | "내가 이 조직을 돌아가게 한다" | 변화량(↑↓), 해결률% |
| 👑 **원장님** | 목표 달성, 의사결정 성공, 성장 그래프 | "내가 만든 것이 지속된다" | 목표 게이지, 타임라인 |
| 👨‍👩‍👧 **학부모** | 성장 확인, 칭찬 받기, 또래 비교 | "나는 좋은 부모다" | 성장 곡선, 칭찬 알림 |
| 🎒 **학생** | XP/레벨업, 뱃지 수집, 연속 기록, 순위 | "나는 성장하고 있다" | 게이미피케이션 |

## 7.2 도파민 트리거 3가지

```
1️⃣ 예측 보상: "이러면 저렇게 될 거야" → 실제로 됨 → 쾌감
2️⃣ 진행 보상: 진행률 0% → 100% 채워지는 과정 자체가 쾌감
3️⃣ 사회적 보상: 인정, 감사, 비교 우위 → 쾌감
```

# 8. FAQ

## Q: 역할을 바꾸면 데이터가 사라지나요?

**A**: 아니요. 역할은 UI 표시 방식만 변경합니다. 모든 데이터는 동일한 데이터베이스에 저장됩니다.

## Q: Quick Tag에서 위험 신호가 감지되면 어떻게 되나요?

**A**: 자동으로 Risk Queue에 추가되고, n8n 웹훅이 트리거되어 Active Shield가 발동될 수 있습니다.

## Q: Risk Queue의 위험 점수는 어떻게 계산되나요?

**A**: 최근 30일간의 상호작용 데이터를 시간 가중치를 적용하여 계산합니다. 최근 데이터일수록 가중치가 높습니다.

## Q: Active Shield는 무엇을 하나요?

**A**: 긍정 리포트 자동 발송, 담당자 알림, 추가 n8n 워크플로우 실행을 통해 선제적으로 이탈을 방지합니다.

## Q: MVP 모드에서 모든 역할을 전환할 수 있는 이유는?

**A**: 테스트 및 점검을 위해 모든 역할을 자유롭게 전환할 수 있습니다. Production에서는 역할 계층에 따른 제한이 적용됩니다.
