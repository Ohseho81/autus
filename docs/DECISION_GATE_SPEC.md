# 🏛️ AUTUS Decision Gate - 수학적 사양서

> "AUTUS는 판단을 잘하게 만드는 시스템이 아니다. 판단이 필요 없게 만드는 시스템이다."

---

## I. 상태 공간

각 결정 단위 \( d \)는 다음 벡터로 표현된다.

```
d = ⟨K, I, Ct, Cm, A, R⟩
```

| 기호 | 정의 | 범위 |
|------|------|------|
| K | Scale (책임 반경) | {1,...,10} |
| I | 비가역성 점수 | [0,100] |
| Ct | 시간 손실 (hours) | ≥ 0 |
| Cm | 금전 손실 (KRW) | ≥ 0 |
| A | 승인 주체 (역할/권한) | 𝒜 |
| R | 규제/법 제약 집합 | ℛ |

---

## II. 핵심 함수

### 1. 비가역성 함수 (Lock)

```
Lock(d) = { 1  if I ≥ θK
          { 0  otherwise
```

**θK (Scale별 임계치):**

| K | θK | 의미 |
|---|---|------|
| 1 | 90 | 거의 모든 것이 되돌릴 수 있음 |
| 5 | 60 | 이사회 결정 수준 |
| 7 | 40 | 다자 합의 수준 |
| 10 | 10 | 거의 모든 것이 비가역 |

---

### 2. 승인 함수 (Approve)

```
Approve(d) = { 1  if A ≥ K
             { 0  otherwise
```

승인 주체의 권한 레벨이 K 이상이어야 승인 가능.

---

### 3. 결정 종결 조건 (Close)

```
Close(d) = Approve(d) ∧ Lock(d)=1 ∧ ∀r∈R, r(d)=true
```

**⚠️ Close가 참이면 세계는 닫힌다.**

이후 모든 파생은 함수적 자동 전개.

---

### 4. 책임 귀속 (Liability)

```
Liability(d) = A
```

- 승인 주체 = 자동 책임자
- 분산 책임/전가 불가
- 사후 "몰랐다" 불가

## IV. 규제 자동 집행 레이어

### 핵심 원칙

1. **법 위에 서지 않는다**
2. **법을 해석하지 않는다**
3. **법을 실행만 한다**

AUTUS는 **재량 이전(pre-discretion) 레이어**다.

### 집행 순서

```
법률 해석 (인간)
 → 규칙화 (1회)
 → AUTUS 규제 함수 등록
 → 이후 해석 개입 불가
```

### 규제 함수 예시

```typescript
{
  id: 'kr-eft-transfer-limit',
  name: '전자금융거래법 - 이체 한도',
  evaluate: (d) => {
    const limit = d.K >= 4 ? 10_000_000_000 : 100_000_000;
    return d.Cm <= limit;
  },
  violationMessage: '전자금융거래법 이체 한도 초과',
}
```

## V. Fog of War UI

### 원칙: 결과만 남기고 경로는 숨긴다

사용자가 보는 것은 오직:

```
⟨ 결정, 비용, 책임, Lock ⟩
```

### 숨겨지는 것

- Reasoning 경로 ❌
- 시뮬레이션 로그 ❌
- 후보 트리 ❌
- 신뢰도 분해 ❌

### 왜 숨겨야 하는가

Reasoning을 보여주면:
- 재량이 생김
- 협상이 시작됨
- 책임이 흐려짐

**AUTUS UI의 목적:**
- 이해시키는 것 ❌
- 결정하게 만드는 것 ❌
- **되돌릴 수 없게 만드는 것 ✅**

## VI. 조직 재편 상태 기계

AUTUS 도입 조직은 **선택지가 없다.** 아래 5단계를 반드시 순서대로 밟는다.

| Stage | 이름 | 핵심 변화 |
|-------|------|----------|
| 1 | 충격 (Decision Shock) | 결정 속도 급증, 관리자 저항 |
| 2 | 권한 재배치 | 승인자 수 급감, 연봉 구조 변화 |
| 3 | 조직 압축 | 5~7계층 → 2~3계층, 회의 소멸 |
| 4 | 외부 확장 | 내부 최소화, 계약이 코드로 대체 |
| 5 | 자율 봉인 조직 | 인간은 결정하지 않음, 상태 기계화 |

### 직무 재정의

| 구분 | 정의 | 권한 | 책임 |
|------|------|------|------|
| Operator | Fact 입력 담당 | ❌ | ❌ |
| Owner | 승인·책임 담당 | ✅ | ✅ |
| System | 집행·기록·전파 | ❌ | ❌ |

**"매니저"라는 직무는 구조적으로 불가능해진다.**
