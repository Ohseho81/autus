# 🏛️ AUTUS CONSTITUTION v1.0

> "원칙이 없는 시스템은 무작위한 결과를 낳는다" - Ray Dalio

---

## 📜 제1조: 존재 목적

**AUTUS는 인간 관계의 물리적 법칙을 발견하고, 이를 통해 사용자의 시간과 에너지를 최적화하는 시스템이다.**

```
Mission: 관계의 ROI를 측정 가능하게 만든다
Vision: 모든 인간관계가 상호 가치를 창출하는 세상
```

---

## 📐 제2조: 핵심 원칙 (Principles)

### P1. 측정 가능성 (Measurability)
> "측정할 수 없으면 관리할 수 없다"

- 모든 관계는 정량화 가능해야 한다
- 모든 행동은 기록되어야 한다
- 모든 결과는 검증 가능해야 한다

### P2. 물리 법칙 준수 (Physics First)
> "인간관계도 물리 법칙을 따른다"

- 에너지 보존: 투입한 시간 = 얻는 가치 (장기적)
- 엔트로피: 관리하지 않은 관계는 소멸한다
- 시너지: 1+1 > 2가 가능하다 (조건부)

### P3. 데이터 기반 의사결정 (Data-Driven)
> "직관이 아닌 데이터로 결정한다"

- 감정적 판단 최소화
- 역사적 패턴 분석
- 예측 모델 활용

### P4. 극단적 투명성 (Radical Transparency)
> "진실이 불편해도 진실을 본다"

- 부정적 시너지도 표시한다
- 에너지 뱀파이어를 식별한다
- 결과를 숨기지 않는다

### P5. 단순함 (Simplicity)
> "복잡함은 버그의 온상이다" - Steve Jobs

- 핵심 기능만 유지한다
- UI는 직관적이어야 한다
- 코드는 읽기 쉬워야 한다

---

## ⚖️ 제3조: 핵심 공식 (Laws)

### Law 1: SQ (Synergy Quotient)
```
SQ = (Mint - Burn) / Time × Synergy_Factor

Where:
- Mint: 창출한 가치 (₩)
- Burn: 소모된 가치 (₩)
- Time: 투입 시간 (분)
- Synergy_Factor: 협업 시너지 계수
```

### Law 2: BaseRate (개인 기준선)
```
BaseRate = 우선순위:
  1) SOLO 이벤트 (혼자 달성) ≥ 2개
  2) ROLE_BUCKET 이벤트 ≥ 2개
  3) ALL 이벤트 (fallback)
```

### Law 3: TeamScore (팀 점수)
```
TeamScore = Σ(개인SQ) + γ×(PairBonus + 0.6×GroupBonus) - BurnPenalty
```

### Law 4: Entropy (엔트로피)
```
Entropy = Burn / Mint

- < 15%: 건강
- 15-25%: 주의
- > 25%: 위험
```

---

## 🚫 제4조: 금지 사항 (Prohibitions)

1. **데이터 조작 금지**: 결과를 왜곡하지 않는다
2. **감정적 예외 금지**: 공식을 임의로 수정하지 않는다
3. **복잡성 추가 금지**: 필요 없는 기능을 만들지 않는다
4. **측정 회피 금지**: 불편한 데이터도 기록한다

---

## 🔄 제5조: 피드백 루프 (Feedback Loop)

```
측정 → 분석 → 결정 → 행동 → 측정 (반복)

주기:
- 일간: 행동 기록
- 주간: KPI 분석, 파라미터 튜닝
- 월간: 역할 재평가
- 분기: 원칙 검토
```

---

## 🎯 제6조: 성공 지표 (Success Metrics)

| 지표 | 목표 | 측정 주기 |
|------|------|----------|
| 순수익 (Net) | 월 +10% | 주간 |
| 엔트로피 | < 15% | 주간 |
| 역할 커버리지 | 100% | 주간 |
| 시너지 평균 | > 0 | 주간 |
| 사용자 DAU | MVP: 100 | 일간 |

---

## 📝 제7조: 개정 절차

1. 원칙 변경은 데이터 기반 근거 필요
2. 최소 4주 데이터 수집 후 검토
3. 변경 시 DECISIONS.md에 기록
4. 변경 이유와 예상 효과 명시

## 📜 제1조: 존재 목적

**AUTUS는 인간 관계의 물리적 법칙을 발견하고, 이를 통해 사용자의 시간과 에너지를 최적화하는 시스템이다.**

```
Mission: 관계의 ROI를 측정 가능하게 만든다
Vision: 모든 인간관계가 상호 가치를 창출하는 세상
```

---

## 📐 제2조: 핵심 원칙 (Principles)

### P1. 측정 가능성 (Measurability)
> "측정할 수 없으면 관리할 수 없다"

- 모든 관계는 정량화 가능해야 한다
- 모든 행동은 기록되어야 한다
- 모든 결과는 검증 가능해야 한다

### P2. 물리 법칙 준수 (Physics First)
> "인간관계도 물리 법칙을 따른다"

- 에너지 보존: 투입한 시간 = 얻는 가치 (장기적)
- 엔트로피: 관리하지 않은 관계는 소멸한다
- 시너지: 1+1 > 2가 가능하다 (조건부)

### P3. 데이터 기반 의사결정 (Data-Driven)
> "직관이 아닌 데이터로 결정한다"

- 감정적 판단 최소화
- 역사적 패턴 분석
- 예측 모델 활용

### P4. 극단적 투명성 (Radical Transparency)
> "진실이 불편해도 진실을 본다"

- 부정적 시너지도 표시한다
- 에너지 뱀파이어를 식별한다
- 결과를 숨기지 않는다

### P5. 단순함 (Simplicity)
> "복잡함은 버그의 온상이다" - Steve Jobs

- 핵심 기능만 유지한다
- UI는 직관적이어야 한다
- 코드는 읽기 쉬워야 한다

---

## ⚖️ 제3조: 핵심 공식 (Laws)

### Law 1: SQ (Synergy Quotient)
```
SQ = (Mint - Burn) / Time × Synergy_Factor

Where:
- Mint: 창출한 가치 (₩)
- Burn: 소모된 가치 (₩)
- Time: 투입 시간 (분)
- Synergy_Factor: 협업 시너지 계수
```

### Law 2: BaseRate (개인 기준선)
```
BaseRate = 우선순위:
  1) SOLO 이벤트 (혼자 달성) ≥ 2개
  2) ROLE_BUCKET 이벤트 ≥ 2개
  3) ALL 이벤트 (fallback)
```

### Law 3: TeamScore (팀 점수)
```
TeamScore = Σ(개인SQ) + γ×(PairBonus + 0.6×GroupBonus) - BurnPenalty
```

### Law 4: Entropy (엔트로피)
```
Entropy = Burn / Mint

- < 15%: 건강
- 15-25%: 주의
- > 25%: 위험
```

---

## 🚫 제4조: 금지 사항 (Prohibitions)

1. **데이터 조작 금지**: 결과를 왜곡하지 않는다
2. **감정적 예외 금지**: 공식을 임의로 수정하지 않는다
3. **복잡성 추가 금지**: 필요 없는 기능을 만들지 않는다
4. **측정 회피 금지**: 불편한 데이터도 기록한다
