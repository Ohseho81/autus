# 🔥 올댓바스켓 현실 점검 (팩폭)

## 현재 상태: UI 목업 (가짜)

### ❌ 가짜인 것들 (alert만 뜨는 것)

| 기능 | 현실 | 문제점 |
|------|------|--------|
| 관리자 지정 | `alert()` | 실제 저장 안됨, 새로고침하면 사라짐 |
| 강사 지정 | `setState()` | 메모리에만 존재, DB 연동 없음 |
| 외부 시스템 연결 | `alert("연결 완료")` | 실제 API 연동 0% |
| 원장 피드백 | `alert("전송됨")` | 어디로? 누구에게? 실제 전송 없음 |
| 업무 체크 | `setState()` | 새로고침하면 초기화 |
| 인사이트 | 하드코딩된 더미 | AI 분석 없음, 그냥 문자열 |
| 영상 촬영 | `alert("업로드됩니다")` | 실제 카메라/업로드 없음 |
| Video Factory | 파일만 존재 | 실제 호출하는 곳 없음 |
| Shadow Learning | 코드만 존재 | 실제 학습 로직 없음 |

### ⚠️ 반쪽짜리

| 기능 | 현실 |
|------|------|
| Supabase 연동 | try-catch로 항상 더미 데이터로 fallback |
| 승인 카드 | UI만 있고 실제 저장/알림 없음 |
| 역할 구분 | 로그인 없이 버튼으로 역할 선택 (보안 0) |

## 진짜 문제점

### 1. 데이터 흐름이 없다
```
원장 → 관리자 → 코치
  ↑_______________↓
```
이 흐름이 코드로 구현되지 않음. 각 화면이 독립적으로 존재.

### 2. 상태 관리 없음
- 전역 상태 관리 없음 (Context, Redux, Zustand 등)
- 각 컴포넌트가 자기 state만 관리
- 화면 전환하면 데이터 소실

### 3. 실제 비즈니스 로직 없음
```javascript
// 현재 코드
const handleSendFeedback = (feedbackId) => {
  alert("원장님께 전송됨"); // 끝?
};

// 있어야 하는 코드
const handleSendFeedback = async (feedbackId) => {
  await supabase.from('feedbacks').insert({...});
  await sendPushNotification(ownerId, {...});
  await recordEvent('feedback_sent', {...});
};
```

### 4. 인증/권한 없음
- 누구나 원장 버튼 누르면 원장
- 실제 서비스에서는 치명적

### 5. Video Factory 미연동
- `video-factory/index.js` 존재
- 하지만 CoachView에서 import조차 안함
- 촬영 버튼 → alert → 끝
