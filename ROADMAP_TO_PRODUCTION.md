## 🚀 상용화 작업 리스트

### Phase 1: 핵심 기능 완성 (1-2주)

#### 1.1 백엔드 안정화
- [ ] **데이터 영속성**: SQLite/PostgreSQL 연동
  - 현재: In-memory StateStore
  - 필요: 세션 간 데이터 유지
- [ ] **에러 핸들링**: 전역 예외 처리 미들웨어
- [ ] **입력 검증**: Pydantic 모델 강화
- [ ] **Rate Limiting**: API 호출 제한

#### 1.2 프론트엔드 완성
- [ ] **오프라인 모드**: Service Worker 캐싱
- [ ] **반응형 UI**: 모바일 최적화 (터치 제스처)
- [ ] **접근성**: ARIA 레이블, 키보드 네비게이션
- [ ] **로딩 상태**: 스켈레톤 UI, 프로그레스 표시

#### 1.3 테스트 커버리지
- [ ] **단위 테스트**: 80% 이상 커버리지
- [ ] **E2E 테스트**: Playwright/Cypress
- [ ] **성능 테스트**: 부하 테스트 (Locust)

---

### Phase 2: 보안 및 인증 (1주)

#### 2.1 사용자 인증
- [ ] **로컬 인증**: 패스워드 해시 (bcrypt)
- [ ] **세션 관리**: JWT 또는 세션 쿠키
- [ ] **OAuth2**: Google/Apple 로그인 (선택)

#### 2.2 데이터 보안
- [ ] **암호화**: AES-256 로컬 데이터 암호화
- [ ] **HTTPS**: TLS 인증서 설정
- [ ] **CORS**: 허용 도메인 제한
- [ ] **CSP**: Content Security Policy 헤더

#### 2.3 개인정보 보호
- [ ] **데이터 최소화**: 필요한 데이터만 저장
- [ ] **삭제 기능**: 사용자 데이터 완전 삭제
- [ ] **내보내기**: JSON 포맷 데이터 내보내기

---

### Phase 3: 배포 인프라 (1주)

#### 3.1 백엔드 배포
- [ ] **Docker 컨테이너화**
  ```dockerfile
  FROM python:3.11-slim
  WORKDIR /app
  COPY requirements.txt .
  RUN pip install -r requirements.txt
  COPY . .
  CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8001"]
  ```
- [ ] **클라우드 배포**: Railway / Render / Fly.io
- [ ] **환경 변수 관리**: .env 분리

#### 3.2 프론트엔드 배포
- [ ] **정적 호스팅**: Vercel / Netlify / GitHub Pages
- [ ] **CDN 설정**: 정적 자산 캐싱
- [ ] **PWA 매니페스트**: 앱 설치 지원

#### 3.3 CI/CD 파이프라인
- [ ] **GitHub Actions**: 자동 테스트 + 배포
- [ ] **스테이징 환경**: 프로덕션 전 테스트
- [ ] **롤백 전략**: 버전 관리

---

### Phase 4: 모니터링 및 운영 (1주)

#### 4.1 로깅
- [ ] **구조화된 로깅**: JSON 포맷 로그
